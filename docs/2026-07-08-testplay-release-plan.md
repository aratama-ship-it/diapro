# テストプレイ配布 計画メモ（2026-07-08）

別の人にテストプレイしてもらうための配布計画。まだ実行せず、方針の下書き。

## 前提（現状）
- 完全な静的アプリ（HTML/CSS/JSのみ・ビルド不要）。外部依存は Google Fonts（CDN＝要ネット）だけ。
- localhost/file:///絶対パス等の“ホスティングで壊れる参照”は無し → 静的ホストにそのまま載る。
- git管理済み（ローカルmain・リモート未設定）。直近変更は未コミット、design/は未追跡。
- モバイルファースト設計 → テスターはスマホで触るのが理想。

## 配布方法の候補（推奨順）
### A. GitHub Pages（継続テスト向き・本命）
- 手順: GitHubでリポジトリ作成 → `git remote add` して push → リポジトリ設定でPages有効化 → `https://<user>.github.io/<repo>` を共有。
- 長所: URL1本でスマホからすぐ遊べる。更新は push で自動再デプロイ。無料。
- 注意: 無料でPagesを使うと**リポジトリは公開**になる（非公開にしたいなら有料 or 別ホスト）。ゲームのテストなら公開でも実害は少ないが要確認。

### B. Netlify Drop（一番速い・単発向き）
- 手順: https://app.netlify.com/drop にフォルダをドラッグ＆ドロップ → 即URL発行。GitHub不要・アカウント無しでも可。
- 長所: 最速。git知識不要。
- 短所: 更新のたび再ドロップ（サイトを“claim”すれば同じURLで更新可）。

### C. Cloudflare Pages / Vercel
- Bと同様の静的ホスト。GitHub連携すると更新が楽。好みで。

### D. zipで送る（非推奨）
- index.htmlを開けば動くが、スマホで開きにくい・更新の再送が面倒。モバイルテストには不向き。

**推奨**: 継続的にテスト＆更新するなら **A（GitHub Pages）**。とにかく今すぐ1回試すだけなら **B（Netlify Drop）**。

## 公開前の準備（テスター向けビルドの整備）
1. **未コミット変更をコミット**（現状をひとまとめに）。
2. **開発用表示を隠す**（テスターに見せない）:
   - 右側の DEV PARAMS パネル（`.dev-panel`）… デスクトップのテスターに見える。予想スコア/ミス率など内部値が出るので隠す。
   - 大会結果の `🔧DEV 実力→実スコア…` の不振理由（`.dev-reason`）。
   - 対応案: `?dev=1` のときだけ表示、等のフラグ化（自分の確認用に残しつつテスターには非表示）。
3. **バージョン表示を追加**（例: タイトル画面かフッターに `v0.9-test1`）。フィードバックがどのビルドのものか分かるように。
4. （任意）**Google Fontsをローカル同梱**すればオフラインでも崩れない。今は未対応でも可（要ネット）。

## フィードバック回収
- 方法候補: Googleフォーム（構造化）／共有メモ／チャットで口頭。おすすめは短いGoogleフォーム。
- 聞く項目案:
  - 端末・ブラウザ（例: iPhone Safari）
  - 何をして何が起きたか（バグ）＋できればスクショ
  - 難易度の体感（簡単/難しい、大会は勝てそうか）
  - 操作性（ボタン届く/文字読める/1画面に収まるか）
  - 面白かった点・分かりにくかった点

## テスターに見てほしいポイント（テスト観点）
- 毎月「何をすればいいか」が直感的に分かるか（練習/勉強/休養→練習メニュー）。
- 難易度・バランス（大会が勝てる/勝てなさすぎないか、卒業ランクの納得感）。
- モバイル表示（各画面が1画面に収まる・タップしやすい・文字サイズ）。
- 主要フロー: つづきから（セーブ/ロード）、怪我・イベント・SNSイベント、リタイア。
- 既知の観察点: 体力が高いのに怪我する挙動（現状は怪我リスク駆動＝仕様、要方針決定）。

## 注意事項
- **セーブ互換**: ゲーム更新で SAVE_KEY を上げると古いセーブは無効化（＝最初から）。更新後は「つづきから」が使えない場合があると周知。
- **キャッシュ**: 更新後はテスターに一度ハードリロードを依頼（ブラウザが旧ファイルを掴むことがある）。将来はファイル名/クエリにバージョンを付けて自動更新化も可。
- **要ネット接続**: Google Fonts CDN 読込のため。オフライン対応するならフォント同梱。

## 実行時の手順（GOが出たら）
1. 未コミット変更をコミット。
2. 開発用表示のフラグ化（dev=1のみ表示）＋バージョン表示追加。
3. ホスト選択:
   - GitHub Pages: リポジトリ作成後、remote追加→push→Pages有効化（リポジトリ作成とアカウント操作はユーザー、`git`操作と設定手順は補助可能）。
   - Netlify Drop: フォルダをドラッグ（ユーザー操作、手順は案内）。
4. 発行URLをスマホで実機確認 → テスター＋フィードバックフォームを共有。
