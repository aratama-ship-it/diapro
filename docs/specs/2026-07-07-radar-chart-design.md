# 技術レーダーチャート（RC）設計書

- 作成日: 2026-07-07
- 対象: ディアボロ選手育成アプリ（app-dev/diabolo-trainer）
- ステータス: 設計（承認済み → 実装計画へ）

## 1. 目的

各ジャンル（1D水平/1D垂直/2D/3D以上）の3ステータス（難易度・新奇性・操作安定度）を、
3軸＝三角形のレーダーチャートで直感的に見えるようにする。得意・苦手の「形」が一目で分かるようにし、
育成の手応えと方針判断を助ける。

## 2. 方針（開発段階の情報量優先）

- **現状の数値テーブル（skillTable）は置き換えず維持**する。その下にレーダーの2×2グリッドを**追加**する。
  開発段階はブラウザで多くの情報を見ながらフィードバックしたいため、情報は最大限残す。
- **レイアウトの厳密な作り込みは最終段階で行う**。本タスクは機能（正しく形が描ける）を優先する。

## 3. 表示仕様

### 3.1 1ジャンル分のレーダー（三角形）

- 3軸の配置: **難易度＝上（頂点）／新奇性＝左下／操作安定度＝右下**。
- 背景: 最大値(100)の薄い三角の外枠＋グリッド（例: 25/50/75/100の同心三角、淡いグレー）。
- 現在値: 各軸の値(0-100)を半径に線形マップし、3点を結んだ塗りポリゴン（半透明ティール＋輪郭線）。
- 各頂点に略ラベル＋数値: 「難27」「新24」「操15」。
- カードのタイトル: ジャンル名＋習熟平均（例: 「1D水平軸 22」）。習熟平均=genreAvg（解禁閾値20の目安）。

### 3.2 ロックジャンル（TR未解禁）

- 塗りポリゴンは描かず、薄い背景三角のみ＋「🔒 未解禁」表示。
- タイトルはジャンル名のみ（平均は0のため省略可）。

### 3.3 レイアウト

- 4ジャンルを2×2グリッドに配置（GENRES順: 1D水平→1D垂直→2D→3D以上）。
- 技術テーブルの直下に配置する。

## 4. コンポーネント設計

app.js はロード時に `document` を参照するため Node の require ができない。座標計算の純関数は
**Node で単体テストできるよう別モジュール `js/radar.js`** に置き、SVG/DOM生成は app.js に置く。

### 4.1 js/radar.js（新規, DOM非依存・DT.radar）

- `radarPoint(value, axisIndex, cx, cy, radius)`: 純関数。値(0-100)と軸番号(0=上,1=左下,2=右下)から
  SVG座標 {x, y} を返す。角度は 上=-90°, 左下=150°, 右下=30°（度→ラジアン変換, x=cx+r'cosθ, y=cy+r'sinθ）。
  r' = radius × clamp(value,0,100)/100。既存モジュールと同じIIFEで `DT.radar = { radarPoint }` を公開。
- index.html の script 読み込みに `js/radar.js` を追加（contest.js等と同じ並び）。

### 4.2 app.js（DOM依存）

既存の `el()` はHTML要素用のため、SVG用に名前空間付き生成ヘルパーを追加する。

- `svgEl(tag, attrs, children?)`: `document.createElementNS('http://www.w3.org/2000/svg', tag)` を使い、
  属性を setAttribute で付与、子要素を append して返す。
- `genreRadar(genreId, cell, avg, unlocked)`: 1ジャンル分のカード(div)を返す。中にタイトル＋svg。
  svgは viewBox 固定（例 100×100）で、背景三角・グリッド・（解禁時のみ）値ポリゴン・頂点ラベル/数値を描く。
  座標は `DT.radar.radarPoint(...)` を使う。
- `skillRadarGrid(skills, state)`: 4ジャンル分の genreRadar を 2×2 グリッドのdivに束ねて返す。
  解禁判定は `DT.contest.isGenreUnlocked({ skills: skills }, genreId)`、平均は `DT.contest.genreAvg({ skills: skills }, genreId)`
  （isGenreUnlocked/genreAvg は state のうち skills しか見ないため、`{skills}` を渡せばメイン画面・スカウト画面の
  どちらでも同じ関数で判定できる）。

## 5. 組み込み箇所

- **メイン画面** `renderMain`: `#main-stats` の技術テーブルの後ろに `skillRadarGrid(state.skills, state)` を追加。
- **スカウト画面** `renderCreate`: 同様に技術テーブルの後ろにレーダーグリッドを追加（生成直後の選手の形を見られる）。
  renderCreate が表示中の仮キャラは `skills` を持つので、`skillRadarGrid(cand.skills, cand)` の形で
  同じ関数をそのまま使える（`skillRadarGrid` は内部で `{skills}` を組んで判定するため cand 全体でも skills だけでも可）。

## 6. 配色（既存ダークテーマ）

- カード背景 `#23233d`（既存 .card と同系）。
- グリッド線・外枠: 淡いグレー（例 `#444` 半透明）。
- 値ポリゴン: 塗り `rgba(78,205,196,0.35)`（ティール半透明）、輪郭 `#4ecdc4`。
- 頂点ラベル/数値: 明るいグレー〜白。

## 7. テスト

- **単体テスト（新規 tests/test-radar.js）**: `DT.radar.radarPoint` の座標計算。
  例: value=0 は中心(cx,cy)に一致、value=100・軸0(上)は (cx, cy-radius)、
  軸1(左下)と軸2(右下)は x が cx に対して対称・y が等しい、を検証（0.001許容）。
  radar.js は DOM非依存なので `require('../js/radar.js')` で Node 実行可能。
- **preview検証**: 大学スタートで3D以上が🔒、h1dを練習すると三角の「難」頂点等が伸びる、
  技術テーブルとレーダーが併存、コンソールエラーなし。

## 8. 非対象（YAGNI）

- 演技構成（ジャンル非依存）はレーダー対象外。現状の演技構成バーは維持。
- レーダーのアニメーション、ホバー詳細、レイアウトの最終的な作り込み（最終段階で別途）。
- 4ジャンルを1つに統合した多角形表示（各ジャンル独立の三角で行く）。
