# カード中央イラスト・簡易ガイドライン（Lite）

最初からガチガチに縛らず、**まず1〜2枚作り、良ければそれを基準に作風を固定していく**方針。
このメモは最低限の枠だけ。詳細版は [2026-07-16-card-art-prompt-spec.md](2026-07-16-card-art-prompt-spec.md)（参照用に温存。今は未適用）。

## 最優先の共通物理仕様

ディアボロ、紐、スティック、人物の3D方向は、カードの画風より先に [ディアボロ画像生成・物理／形状制約](../../../../character-consistency-refs/diabolo-physics-constraints.md) を適用する。構図やポップさが良くても、同仕様のハード失格に1つでも該当する画像は採用しない。

特に、本体は「外側へ開くお椀状カップ2個＋両カップ間の中央軸1本」である。紐は2カップ間に露出した中央軸だけに接触し、カップの外側・外周・表面・中心ハブ・内壁・空洞から出たり、カップを貫通したりしてはならない。軸接触が見えない場合は本体・人物・両手・カメラの向きを調整する。

## 守ってほしいのはこれだけ
1. **アスペクト比**：横長 **約2:1**（アート枠は DOM 約280×150 / 書き出し 592×300）。
   16:9 や 3:2 の横長でもOK（`object-fit: cover` で表示するため、上下が少しトリミングされる前提で中央に余白を）。
2. **色**：自由。カードごとに何色を使ってもよく、クール基調へ固定しない。現在の作風候補では、黄・赤・ピンク・シアン・青などを使ったキャッチーでポップな高彩度配色を優先する。
3. **被写体**：ディアボロ（2つの椀＋中央軸＋2本のスティック＋紐）が主役で分かること。
4. **文字を入れない**：画像内にロゴ・文字・数字・ウォーターマークを描かない（カード側で文字を載せるため）。

## 縛らないこと（自由でOK）
- 画風（イラスト/半写実/アニメ等）、タッチ、構図、エフェクト、mood、背景の作り込み具合。ただし現在の第一候補は、太い輪郭、セル塗り、色面、ハーフトーンを使うキャッチーなポップ柄。
- ディアボロの色の完全一致、人物の有無など。ただしディアボロと紐の形状・接触・遮蔽・個数・3D方向は共通物理仕様に従い、自由項目には含めない。
→ 1〜2枚で「これ」という方向が出たら、その特徴を後から言語化して固定する。

## `mx_E_showman`「お祭り好き」の今回限定例外

- 人物を描いてよい。
- 通常のディアボロ用ハンドスティックの代わりに、同形の太鼓のバチ2本を使ってよい。
- 紐の端点は、各手から遠い側にあるバチの打撃先端へ1つずつ接続する。手元、側面、途中、尻側から出してはならない。
- バチ例外でも、本体1個、カップ2個、中央軸1本、バチ2本、連続した紐1本、端点2個、分岐ゼロ、張ったV字、中央軸への実接触、カップ非貫通は維持する。
- 紐は中央軸の下面へ短く接触し、本体が張った紐の上に乗るように見せる。軸の側面・上面へ回り込ませ、巻き付いているように見せない。
- 各バチの全長は、左右のバチ先端間を中央軸下面経由で測った紐全長の30%以下とする。紐1mなら各バチは30cm以下。
- 各手の握り中心は、紐が出るバチ先端から全長の約2/3地点とし、手の後ろへ尻側の約1/3を残す。
- 人物が正面を向く構図では、ディアボロの軸を人物の胴体正面面に対して垂直、すなわちほぼ観察者方向へ向ける。接触点を見せるための最小限のヨー角だけを許容する。

## 非演技状態のカード

- カード内容が手持ち・展示・卒業記念・メダル等で、ディアボロを回していないことを明示する場合は、紐とスティックを省略してよい。
- 手持ち／展示本体は、外側へ開くお椀状カップ2個＋その間の中央軸1本として読み取れること。手指や台座で形を隠し、単一円盤やヨーヨーに見せない。
- 横向きの本体を中央軸で持つ場合は、「狐の手」に近い形で親指と中央の長い中指を向かい合わせて軸をつまむ。親指と小指で軸をつままない。人差し指・薬指・小指は手に近い側のカップの弧の外側（空洞と反対側の凸面）へ添え、空洞内へ入れたりリムを越えて内壁へ差し込んだりしない（詳細は共通物理仕様を参照）。
- 紐やスティックを描く場合は非演技例外を解除し、共通物理仕様の標準演技条件をすべて適用する。
- `sp_unhurt`「無傷の四年間」の初回案は、紐・スティックなしの本体1個を卒業生が手に持つ構図とする。

## 方針：50枚それぞれ固有の絵にする
全50カードを1枚ずつ個別イラストにする（＝カードID単位）。差し込み口はID基準に配線済み。
未登録のカードは従来どおり属性TypeのSVG表示のまま（表示・保存・シェア対応済み）。

## 差し込み方（作った画像の入れ方）
1. 画像を `assets/cards/<カードID>.png` に置く（例：`assets/cards/sp_worlds.png`）
2. `js/app.js` の `CARD_IMAGE` に1行足す（例：`sp_worlds: 'assets/cards/sp_worlds.png',`）
3. 未登録カードはSVGのまま。1枚ずつ差し替わっていく
