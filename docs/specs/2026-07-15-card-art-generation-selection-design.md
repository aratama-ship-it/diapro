# カード中央画像 生成・選択 設計書（事前生成方式）

作成日: 2026-07-15 / 対象: ディアボロ選手育成アプリ / 関連: [トレーディングカード・エンディング設計](./2026-07-15-trading-card-ending-design.md)

## 方針
- カードの**中央メインビジュアル**は、**事前に用意した画像**の中から選ぶ。ランタイム自動生成はしない
  （トークン/コスト/遅延/データ肥大を回避）。
- 選択は「その選手カードの特徴（複数）」を組み合わせ、**最も適した1枚**を選ぶ。
- 特徴を2種に分離して画像の総数を現実的に抑える：
  - **ベース画像を分ける軸**（少数・視覚的に効く特徴）＝事前生成する枚数を決める。
  - **それ以外**＝コードで上乗せ（フレーム/キラ/★/メダル/数値/章/テキスト）。画像は増やさない。

## 特徴（Features）と抽出元
| 特徴 | 抽出元 | 値域 |
|---|---|---|
| ランク（レア度） | `e.rank` | S/A/B/C/D/E |
| 世界王者 | `worldsWin` | bool |
| 全日本王者 | `ajdcOverallWin` | bool |
| 勝ちっぷり | `state.results` の優勝数・表彰台(≤3)率 | 常勝 / 勝ち越し / 苦戦（3段） |
| 属性Type | 4系統(難易/新奇/操作/構成)のargmax・均衡 | 高難度 / イノベーター / テクニシャン / ショーマン / 万能（5） |
| 最強ジャンル | ジャンル別習熟のargmax | 1DH / 1DV / 2D / 3D+（4） |
| 経歴 | `state.background` | 大学/高校/中学（3。幼少期は2026-07-15廃止） |

## ベース画像を分ける軸（＝事前生成する枚数）★要確定
**推奨案：属性Type(5) × ランク帯(3)** ＝ 15枚 ＋ 特別（世界王者など）数枚。
- ランク帯 = 高[S/A] / 中[B/C] / 低[D/E]。
- Type＝中央ディアボロ/構図の「形・作風」、ランク帯＝舞台の格（照明/観客/豪華さ/発光）。
- 「最強ジャンル」「勝ちっぷり」「称号」「経歴」は**コード上乗せ**で表現し、画像は増やさない。
- 総数はここで決まる。まず 15+α で開始、後から季節/イベント/ジャンル別の限定アートを追加可能。
- 別案：Type×ランク帯×最強ジャンル(4) = 60枚（リッチだが生成コスト大）。要判断。

## コードで上乗せする要素（ベース画像は共通のまま可変にできる）
- レア度のキラ/ホロ・フレーム色・★数（S=虹UR … E=N、退学=特殊）
- 能力CP / 通算pt の数値、4系統ステータス
- 実績メダル（🏆世界王者 / 🥇全日本V / 優勝回数 / 表彰台率）
- 最強ジャンルの章（エンブレム/カラーアクセント）
- 選手名・称号・経歴・No.

## シミュレーションで決めること（用意する画像リストの確定）
多数の通しプレイをシミュレートし、特徴プロファイルの**出現分布**を測る。
- 目的：
  1. よく出るプロファイル＝優先して画像を用意。ほぼ出ない組合せ＝汎用フォールバックで済ませる。
  2. 各特徴が「区別できるほど」出現するか検証（例：どのTypeも十分出るか／偏っていないか）。
  3. ベース軸を Type×ランク帯 で十分か、ジャンルも軸に要るかを分布から判断。
- 方法：戦略を複数用意して偏りを避ける（弱点補強 / 一点特化 / バランス育成）。各 N=数百回。
- 出力：`(Type, ランク帯, 最強ジャンル, 勝ちっぷり)` のヒートマップ/頻度表。

## 画像生成の共通ブリーフ（プロンプト方針）
- 全体トーン：クールでシャープ（スチール/ブルー/シアン基調）。中央に**大きなディアボロをドーンと**。**人物は描かない**。
- Type別の作風：
  - 高難度型＝鋭角・トゲ状・攻撃的シルエット
  - イノベーター型＝異形・非対称・不規則な造形
  - テクニシャン型＝精緻な同心円・左右対称・端正
  - ショーマン型＝装飾的・華やか・光の演出多め
  - 万能型＝王道・バランスの取れた標準形
- ランク帯別の格：高＝荘厳な舞台・強い発光・粒子、中＝標準、低＝シンプルで静か。
- 画角/比率：カードのアート枠に合わせる（比率は実装枠に合わせて確定）。中央寄せ・周囲に余白。
- **文字はベース画像に入れない**（名前/数値/称号はコード側で重ねる）。
- 命名規約：`art_<type>_<ranktier>.png`（例 `art_technician_high.png`）＋特別 `art_worldchamp.png` 等。

## 選択ロジック（実装接続）
```
pickCardArt(features):
  if worldsWin: return 'art_worldchamp'         # 特別を最優先
  base = 'art_' + type + '_' + rankTier          # 例 art_sh/technician/high
  return existing(base) ? base : 'art_fallback'   # 用意が無ければ汎用へ
```
- 優先順位付きルールで最適1枚を返し、該当画像が無ければ汎用フォールバック。
- 特徴→画像の対応表（featureKey → imageId）をデータ化して管理。

## アート図鑑（アーカイブ・コレクション）【2026-07-15 追加】
カード本体（保存10枚上限）とは別に、**タイトル付きアート画像を永続コレクションする図鑑**を設ける。
"カードを溜める"のではなく"アート作品を集める"のが長期モチベーション。

- **各アートに作品タイトルを付ける**（例: PRISM／CONCENTRIC…＋和名。画像とタイトルで1作品扱い）。
- **解禁条件 = その特徴プロファイルで卒業すること**（例: テクニシャン型×高ランクのアートは、
  その組合せのカードを一度出すと図鑑に永続登録される）。
- **保存は超軽量**：図鑑にはアートID＋初解禁日のみ記録（画像はコード/アセット側にあるので
  localStorageは数十バイト/作品。10枚上限の対象外・上限不要）。
- **図鑑UI**：タイトル画面 or「これまでの記録」から開く。未解禁はシルエット＋「???」表示。
  解禁済みはタップで全画面鑑賞（タイトル・初解禁日・出したカードの要約つき）。
- **周回動機の設計**：Type(5)×ランク帯(3)=15＋特別（世界王者/全日本V/退学 等）で
  「全部で N 作品」と明示し、コンプ率（12/18 など）を表示する。
  - 世界王者アート（出現率0.4%）が最後のコンプ目標になる想定。
- カード（10枚上限・押し出しあり）は"戦績アルバム"、図鑑は"画集"として役割分担する。
