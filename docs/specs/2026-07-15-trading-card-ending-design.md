# トレーディングカード・エンディング 設計書

作成日: 2026-07-15 / 対象: ディアボロ選手育成アプリ

## 概要
卒業（またはGAME OVER）時、育てた選手を1枚の**トレーディングカード**に変換して表示する。
「ステータスがそのままカードの性能になる」のがコンセプト。パック開封風の演出で登場させ、
記録に保存＋画像ダウンロードにも対応する。

## ビジュアルの方向性
- **クール（冷たい）トーン**：スチールブルー/ダークスレート/シアンを基調にしたシャープな配色。
  暖かい金/クリームは"レアリティの差し色"としてのみ使う（S=虹ホロ、A=金ホロ）。
- **キャラクター（顔パーツ選択など）は作らない**：実装コストが高いため見送り。
- **中央に大きなディアボロを主役として配置**（キャラの代わり）。コード内SVGで作り置きした数種の
  ディアボロ・アートから1つを選んで表示（毎回自動生成しない＝トークン/データ節約）。

## カード構成要素 → データ対応
| カード要素 | 元データ |
|---|---|
| レアリティ（枠色・キラ・★数） | 卒業ランク S/A/B/C/D/E（evaluate） |
| 選手名 | `state.name` |
| 称号（epithet） | `e.title` |
| **中央アート（大きなディアボロ）** | 作り置きSVGバリエーションから選択（選択軸は下記） |
| 属性（Type） | 4系統の偏りで自動判定 |
| ステータス（4系統・数値/小バー） | 各ジャンル平均＋演技構成 |
| 能力CP（大数字） | 能力の総合値（下記式） |
| 通算pt（別枠） | `e.totalPoints` |
| 実績メダル | 世界王者/全日本V 等（state.results） |
| 経歴・No. | `state.background` ＋ 獲得順の通し番号 |

※旧案の「4軸レーダーを中央アートにする」は取りやめ。中央はディアボロ・アート、ステータスは数値＋小バーで表示する。

## レアリティ（卒業ランク連動）＝キラ/ホロの度合い
ランクが上がるほど「キラ加工」を強くする（トレカのノーマル→レア→キラ→ウルトラレアの発想）。
| ランク | レア度 | ★ | 加工イメージ |
|---|---|---|---|
| **S** | ウルトラレア(UR) | ★6 | 全面 虹ホログラム＋強い光沢・パーティクル |
| **A** | スーパーレア(SR) | ★5 | 金ホロ枠＋光沢（キラカード） |
| **B** | レア(R) | ★4 | 銀の部分ホロ・軽い艶 |
| **C** | ノーマル+(銅) | ★3 | 銅色フレーム・艶控えめ |
| **D** | ノーマル | ★2 | 艶なし |
| **E** | ノーマル | ★1 | 艶なし |
| 退学 | 特殊 | ★0 | グレー/ヒビ・"GAME OVER" |
- キラ加工＝CSSのグラデ/ホログラム風シャイン＋（Phase2で）光のスイープアニメ。レア度が高いほど派手に。
- 世界王者・AJDC総合優勝は evaluate で既に S 扱い（ランク連動なので特別枠は作らない）

## 数値仕様
- **能力CP** = round( 13ステータス合計 / 1.3 ) → 0〜1000（12マス＋演技構成＝各0-100、満点1300を1000に正規化）
- **通算pt** = `e.totalPoints`（そのまま。実績の集大成）
- **4系統バー**（各0-100）: 難易度/新奇性/操作安定度＝各ジャンル平均、構成＝`state.composition`
- **属性（Type）**: 4系統の (最大-最小) ≤ 8 なら「万能型」。それ以外は最大の系統で判定
  - 難易度最大 → 高難度型 ／ 新奇性最大 → イノベーター型 ／ 操作最大 → テクニシャン型 ／ 構成最大 → ショーマン型
- **No.** = 記録保存時の獲得順連番（プレイヤーが積み上げた"何人目の卒業生か"）

## 中央アート（ディアボロ）バリエーション ＝ 作り置き【確定】
- **方式**：コード内SVGでハンドメイドした数種を用意。画像ファイル/AI生成は使わない（軽量・データ肥大なし）。
- **選択軸＝属性（Type）**：プレイスタイルごとに"署名ディアボロ"が変わる（5種）。
  | 属性 | 判定 | ディアボロ・アートの方向性 |
  |---|---|---|
  | 高難度型 | 難易度が最大 | 鋭角・トゲ状・攻撃的シルエット |
  | イノベーター型 | 新奇性が最大 | 異形・非対称・不規則な造形 |
  | テクニシャン型 | 操作安定度が最大 | 精緻な同心円・左右対称・端正 |
  | ショーマン型 | 演技構成が最大 | 装飾的・華やか・光の演出多め |
  | 万能型 | 4系統の最大-最小 ≤ 8 | 王道・バランスの取れた標準形 |
- **レアリティ（ランク）は色/キラ加工で上乗せ**：同じ型でもS＝虹ホロに光る（Type=形、Rank=輝き）。
- 将来、季節/イベント限定アートを `ART[id]` を増やすだけで拡張可能に。

## 演出（パック開封風）
1. エンディング遷移 → カードパック（裏面）が中央に発光しながら登場
2. タップ → 開封アニメ。レア度に応じた発光/パーティクル（S=虹、A=金…）
3. カード表が現れ、能力CP・ステータスがカウントアップ表示
4. カードの下に既存の大会成績表を配置（スクロール）

## 保存・共有
- 記録（RECORDS）に card スナップショットを追加保存
  （name, background, date, rank, title, cp, totalPoints, type, stats{難易/新奇/操作/構成}, medals[], cardNo）
- **保存上限＝1プレイヤー10枚**（画像化でデータ量が増えるため）。
  - 画像(PNG)そのものは localStorage に**保存しない**（データ肥大回避）。カードは軽量な上記データだけ保存し、
    表示・ダウンロード時にその場で描画/PNG化する。
  - 11枚目以降は、既存の記録ロジックに合わせ**通算pt上位10枚を保持**（下位を押し出す）。※「最新10枚」に変えるかは要検討。
  - 現状 `RECORDS_MAX=20` → カード用に 10 へ（または別キーで10管理）。
- 「これまでの記録」モーダルをミニカード一覧化＋タップで拡大表示
- カード画像ダウンロード（カードDOM/SVGをPNG化して保存。スクショ共有用）

## 実装フェーズ分割（提案）
- **Phase 1**: カード本体の描画＋レアリティ＋ステータス（演出は簡易フェードで先出しβ）
- **Phase 2**: パック開封演出（発光・カウントアップ・レア度別エフェクト）
- **Phase 3**: 記録へのカード保存＋一覧のミニカード化＋画像ダウンロード
