# カード中央イラスト用プロンプト仕様書 v0.1

作成日: 2026-07-16  
対象: スマホ育成ゲーム「ディアボロ選手育成」トレーディングカード全50種  
ステータス: **1周目・作風選定用。全50枚への展開前**  
関連: [カード中央画像 生成・選択 設計書](./2026-07-15-card-art-generation-selection-design.md) / [カードカタログ（全50種）](./2026-07-15-card-catalog-50.md) / [トレーディングカード・エンディング設計](./2026-07-15-trading-card-ending-design.md)

> **2026-07-16 現行方針:** 本書のクール配色、人物なし、A/B/C作風固定は比較検討用の旧案であり、現在の生成にはそのまま適用しない。現行の色・人物・ポップ作風は [Liteガイドライン](./2026-07-16-card-art-guidelines-lite.md) を優先する。ディアボロ本体、紐、スティック、人物との3D方向については、画風に関係なく [共通物理仕様](../../../../character-consistency-refs/diabolo-physics-constraints.md) を最優先する。

## 0. この文書の役割

この文書は、外部の画像生成AI（ChatGPT Images / gpt-image系）で中央イラストを1枚ずつ作るための、作風・プロンプト・検品・運用の正典である。1周目では作風候補を比較し、推奨案Aを暫定正典として代表5枚だけ実演する。**残り45枚の個別プロンプトは、作風決定とcanon画像の承認後に次周で作成する。**

### 0.1 既存設計との整合

- ランタイム生成は行わず、PNGを事前生成してリポジトリに同梱する。same-origin画像として表示・canvas保存に使う方針は継承する。
- カード名、選手名、ランク、メダル、数値、フレーム、ホロ加工などはコード側で重ねる。中央イラストには入れない。
- 排出判定はカードカタログの `特別15 → 職人5 → ランク×属性30` の優先順を変更しない。
- 旧画像選択設計の `Type × ランク帯 = 15枚 + 特別` は、制作量を抑える案としては有効だが、今回の「全50種に固有の中央イラストを用意する」という要件では最終画像数と一致しない。本仕様では**50カードIDごとの専用PNG**を正とし、旧15枚案は制作途中または画像欠損時のフォールバックとしてのみ残す。
- 実装接続時は現行の `CARD_IMAGE[type]` ではなく `CARD_IMAGE[card.id]` を第一候補にし、未登録時だけ `type` の署名SVGまたは共通画像へフォールバックする。これは次周以降の実装事項であり、本周ではコードを変更しない。

### 0.2 今周で決めること / 決めないこと

| 今周で決める | 次周へ送る |
|---|---|
| 作風候補A/B/C、固定デザイン、固定色、構図、禁止事項、プロンプト組み立て、canon運用、代表5枚 | 作風の最終承認、canon画像の選定、残り45枚の差分プロンプト、50ファイルの最終対応表 |

## 1. 作風の方向性候補

3案とも、人物を描かずディアボロを主役にし、クール配色、中央寄せ、横長、文字なしという不変条件を共有する。違うのはメディウム、線、塗り、光の扱いだけである。

| 比較軸 | A: テクニカルグラフィック | B: 半写実CG | C: セルペイント |
|---|---:|---:|---:|
| 50枚の一貫性 | **5 / 5** | 3 / 5 | 4 / 5 |
| 小サイズの可読性 | **5 / 5** | 3 / 5 | 4 / 5 |
| 素材の高級感 | 3 / 5 | **5 / 5** | 3 / 5 |
| 感情・速度の強さ | 4 / 5 | 4 / 5 | **5 / 5** |
| UIホロとの共存 | **5 / 5** | 3 / 5 | 4 / 5 |
| canon参照への依存 | 低〜中 | **高** | 中 |

点数は本企画の制約に対する相対評価であり、一般的な画力や品質の優劣ではない。

### A. クリーン・テクニカルグラフィック

**狙い:** 精密なスポーツギアを主役にした、プレミアムなテクニカルポスター風の2Dイラスト。ベクターライクな輪郭と限定色の面で、270×150px程度の小さな表示でも一目で読める絵にする。

**長所**

- 単純化した面、線幅、色数を言語で固定しやすく、50枚をまたぐドリフトに最も強い。
- エフェクトの形だけで「鋭さ」「精密さ」「華やかさ」「挫折」を描き分けやすい。
- 小サイズ、角丸クリップ、`object-fit: cover` でも輪郭が潰れにくい。
- カードUIのホロやフレームと競合しにくい。

**短所**

- 写実的な素材感や映画的な豪華さはBより弱い。
- 背景を簡略化しすぎると、50枚がアイコンの色違いに見える。
- 生成結果によっては「SVGアイコン」「インフォグラフィック」寄りになるため、奥行きと光の階調を明記する必要がある。

**固定パレット**

| 役割 | hex | 使用目安 |
|---|---:|---|
| 最暗部 / 外周 | `#050B16` | 15–25% |
| 背景ディープネイビー | `#081A2E` | 30–45% |
| 背景スチールブルー | `#244B69` | 10–20% |
| ディアボロ本体ブルー | `#183F5D` | 15–25% |
| リム / 構造線 | `#5F86A3` | 5–10% |
| 主発光シアン | `#19D9FF` | 5–8% |
| 氷白ハイライト | `#C8F4FF` | 2–5% |
| 補助クールバイオレット | `#7478FF` | 0–4% |

色はアンチエイリアスや発光の中間色を除き、この8色から外さない。バイオレットは伝説・覚醒・ショーマンなどに限り、画面の4%以下にする。

**style preamble たたき台**

```text
横長16:9の、クリーンで精密なテクニカルグラフィック・イラスト。限定されたディープネイビー、スチールブルー、シアン、氷白、少量のクールバイオレットだけを使う。ベクターライクな明瞭な外形、均一で細い構造線、2〜3段階の面塗り、控えめなエアブラシ発光。中央に規格化された同一デザインのディアボロを大きく配置し、2本のハンドスティックと一本の紐を人物や手なしで見せる。背景は抽象的な競技空間。文字、ロゴ、記号、透かし、カード枠は描かない。
```

### B. シネマティック半写実CG

**狙い:** ディアボロを高級スポーツギアとして撮影したような、半写実のハードサーフェスCG / ペインタリー仕上げ。素材の反射と空気感でレアカードらしい高級感を出す。

**長所**

- ポリカーボネート、金属軸、紐の素材差が魅力になり、「実在する道具」として伝わりやすい。
- 光量・被写界深度・粒子でランクやドラマを大きく表現できる。
- 1枚ごとの鑑賞性が高く、図鑑を画集として見せやすい。

**短所**

- カップ形状、反射、レンズ、背景の具体物が世代ごとに変わりやすく、Aよりcanon参照への依存が強い。
- 暗部が小サイズで潰れやすく、派手な光がカードUIのホロと競合しやすい。
- AIがヨーヨー、糸巻き、機械部品、武器へ誤解する可能性が比較的高い。

**固定パレット**

| 役割 | hex | 使用目安 |
|---|---:|---|
| 最暗部 | `#02060D` | 20–30% |
| 背景ネイビー | `#071528` | 25–40% |
| 背景ブルー | `#102F4A` | 10–20% |
| 本体ポリマー | `#1C4967` | 15–25% |
| 金属スチール | `#6D91A8` | 5–10% |
| 主発光シアン | `#16D8FF` | 4–8% |
| 反射ハイライト | `#D5F7FF` | 2–5% |
| 補助バイオレット | `#6672E8` | 0–4% |

**style preamble たたき台**

```text
横長16:9のシネマティックな半写実ハードサーフェスCGイラスト。高級スポーツ用品のキービジュアルのように、深いネイビー空間、スチールブルーの半透明ポリカーボネート、精密な金属軸、細いシアンの発光を描く。固定された同一モデルのディアボロを中央で鮮明にし、左上の面光源、右後方のシアンのリムライト、低い位置の柔らかな反射光を毎回維持する。背景は抽象的で人物も観客もいない。過度なフォトリアル、文字、ロゴ、透かし、カード枠は描かない。
```

### C. アニメ・セルペイント

**狙い:** スポーツアニメの必殺技カットを、人物なしで道具だけに凝縮した2Dセル画調。太さを管理した輪郭、2段影、スピード感のある光跡で感情を強く伝える。

**長所**

- ポーズと光跡で動きが伝わりやすく、タイトルの感情差を大きく出せる。
- Aよりドラマチック、Bより輪郭が明快で小サイズに強い。
- 挫折、挑戦、祝祭、反復など物語系カードと相性がよい。

**短所**

- 「アニメ」の指示から人物、手、擬人化した目や顔が混入しやすい。
- スピード線、稲妻、集中線が増殖しやすく、50枚で画面密度が揺れやすい。
- 線の太さやセル影の段数が変わると、別作品に見えやすい。

**固定パレット**

| 役割 | hex | 使用目安 |
|---|---:|---|
| 輪郭 / 最暗部 | `#030814` | 10–18% |
| 背景ネイビー | `#09172C` | 30–45% |
| 影ブルー | `#123653` | 15–25% |
| 本体中間色 | `#2B6383` | 12–22% |
| 構造線 | `#79A8BE` | 4–8% |
| 主発光シアン | `#27D9FF` | 5–9% |
| 氷白 | `#D7F8FF` | 2–5% |
| 補助バイオレット | `#7278E8` | 0–4% |

**style preamble たたき台**

```text
横長16:9のスポーツアニメ・セルペイント調イラスト。人物の代わりにディアボロだけを主人公として描く。固定された同一モデルを、濃紺の細い輪郭、輪郭内の2段セル影、限定されたスチールブルーとシアンの発光で表現する。線幅と影の段数は全画像で一定。速度線は短く整理し、中央のディアボロのシルエットを隠さない。2本のハンドスティックと一本の紐は見せるが、人、手、顔、擬人化、文字、ロゴ、透かし、カード枠は描かない。
```

### 1.1 推奨: A「クリーン・テクニカルグラフィック」

本シリーズでは、単体の豪華さよりも**50枚を並べたときの同一作品感**、小さな中央パネルでの可読性、毎回の再生成可能性を優先する。そのためAを推奨する。固定形状を崩さず、背景光・軌跡・幾何モチーフの差だけで意味を与える今回のルールとも最も相性がよい。

Bはcanon参照を十分に効かせられる場合の高級感で優れ、Cはドラマ性で優れる。ただしBは素材と形状、Cは線とエフェクトがドリフト源になりやすい。以下のStyle Bibleと代表5枚は、**レビュー用の暫定正典としてAを採用**して記述する。BまたはCを採用する場合も、ディアボロ形状、構図、背景階層、禁止事項、運用フローはそのまま使い、画風・パレット・ライティングの項だけを差し替える。

## 2. 共通スタイル聖典（Style Bible）

### 2.1 固定と可変の境界

| 区分 | 要素 | ルール |
|---|---|---|
| **全50枚で固定** | 画風、線、塗り、色集合、物体モデル、材質、光源方向、カメラ、画角、背景の抽象度、エフェクト上限、出力サイズ | カード固有の都合で変更しない |
| **カードごとに可変** | ポーズ、エフェクト、小物、背景の光、mood | 後述の上限内で、タイトルの意味を1つの主比喩に要約して変える |
| **意味上の例外** | ディアボロの個数 | `双皿の遣い手` は2個、`三連の魔術師` は3個を基本とする。すべてcanonと同じモデルの複製にし、各個体のカップ形状・色・材質は変えない。それ以外は原則1個 |
| **画像外 / UI側** | カード枠、レアリティ色、ホロ、タイトル、選手名、数値、章、メダル | 中央イラストへ焼き込まない |

旧設計ではTypeごとにディアボロ形状を変える案があったが、50枚の一貫性を最優先する本仕様では採用しない。高難度型の「鋭さ」、イノベーター型の「異形」、テクニシャン型の「精密」、ショーマン型の「華」、万能型の「均整」は、**本体の改造ではなく、ポーズ・軌跡・背景光の文法**で表現する。

### 2.2 画風 / メディウム

- クリーン・テクニカルグラフィック。最終成果物はラスターPNGだが、輪郭と色面は精密なベクターイラストのように整理する。
- スポーツギア広告の精度と、抽象的な競技ポスターの躍動感を両立する。
- 写真、油彩、水彩、厚塗り、3Dフォトリアル、コミック網点、ピクセルアートは混ぜない。
- 画面の情報量は「主役1、主比喩1、補助比喩1」まで。装飾の足し算で差別化しない。

### 2.3 レンダリング（線・塗り・質感）

- 外形線は濃紺 `#050B16`、出力1600×900換算で4–6px相当。内部の構造線は2–3px相当。線幅をカードごとに変えない。
- 本体はベース色、影、ハイライトの3段階。滑らかなグラデーションは発光と背景だけに使う。
- カップはサテン調の硬質ポリマー。鏡面、傷、汚れ、木目、布地、過剰な透明感は付けない。
- リムはスチールブルーの硬い縁、軸は氷白からシアンへ落ちる小さな金属光沢。ブランド刻印やネジなどの微細ディテールは描かない。
- 発光はシアンの細い縁光と、半径の小さいソフトグローだけ。白飛びで本体の輪郭を消さない。
- ノイズや紙目は原則なし。必要なら背景にのみ、ほぼ知覚できない均一な粒状感を3%以下で加える。

### 2.4 ディアボロの標準デザイン / 比率

**物体同定:** ディアボロは、左右に開いた2つの深いカップ状の椀が、中央の細い金属軸で連結されたジャグリング道具である。ヨーヨー、ベーゴマ、糸巻き、ダンベル、ボウタイ、宇宙船として描かない。

- 左右のカップは完全な鏡像。同じ直径、深さ、リム厚、色、材質にする。
- 正面に近い側面視で、左右の開口リムが外側、細く絞られたカップ底が中央軸側を向く。
- 本体全体の「軸方向の長さ : カップ外径」は約 `1.35 : 1`。極端に横長、球形、円盤形にしない。
- 各カップの最大外径を100としたとき、中央軸の見かけの直径は約10、左右カップ間の軸の見える長さは約18、リム厚は約6を目安とする。
- カップ外縁は滑らかな円。トゲ、刃、羽、穴、歯車、宝石、顔、目、発光文字を本体に追加しない。
- 本体色は `#183F5D`、影は `#081A2E`、リムは `#5F86A3`、軸の主ハイライトは `#C8F4FF`、縁発光は `#19D9FF` に固定する。
- 2本のハンドスティックは同一の細い直棒。濃紺ボディに短いスチールブルーの先端。画面左右の周辺に置き、手や腕を描かない。
- 紐は一本だけ。細い氷白〜シアンで、2本のスティック間をつなぎ、中央軸の下または周囲を物理的に通る。途中で増殖、分岐、結び目化させない。
- 通常カードは本体1個。複数使いカードでも、1個の本体を多カップ化せず、同じ2カップ構造のディアボロを2個または3個描く。

### 2.5 配色

A案の8色を固定パレットとする。面積配分は概ね、ネイビー系55–65%、本体とスチールブルー25–35%、シアンと氷白8–12%、バイオレット0–4%。

- 暖色は使わない。金・黄・橙・赤・ピンク・茶・クリームはカードUI側のレアリティ表現に任せる。
- タイトル差を色相変更だけで表現しない。同じシアンでも明度、面積、光の向き、密度で差を出す。
- `未完の大器` など暗いカードでも、彩度を落とすのは背景光だけ。本体の基準色は変えない。
- `伝説のディアボリスト` など最上位カードでも、白とシアンの面積合計は15%を超えない。

### 2.6 ライティング

- 固定キーライト: 画面左上10時方向から、やや硬いスチールブルーの面光。
- 固定リムライト: 画面右後方2時方向から、細いシアンの縁光。
- 固定フィル: 下方正面から弱いネイビーの反射光。黒潰れを防ぐが、底面を明るくしすぎない。
- 本体の明暗方向は全画像で固定する。カード固有の背景ビームが増えても、本体のキーライト方向は反転させない。
- 投影影は背景から浮かせるための柔らかな接触影だけ。床や台座は描かない。

### 2.7 被写界 / 構図テンプレート

- カメラは中央正面、望遠寄りの歪みの少ない見え方。魚眼、広角パース、極端な俯瞰・煽りは使わない。
- 基準はディアボロの軸がほぼ画面左右を向く側面視。カード固有のポーズで軸の傾きを `-18°〜+18°` の範囲だけ変える。奥行き方向の回転は `±10°` 以内。
- 主役の中心は画像中心から左右5%、上下6%以内。物語上あえて低くする場合も上下10%以内。
- 通常1個の本体は画面幅の38–48%、画面高の42–58%に収める。周囲に十分なネガティブスペースを残す。
- スティックは左右下寄りを基本とし、先端から画面端まで5%以上空ける。紐と光跡を含む全重要要素はセーフゾーン内に置く。
- 角丸で切られても成立させ、左下は現行の `ART:` ラベルが重なるため低コントラスト・低密度にする。

**共通レイアウト（1600×900マスター）**

| 領域 | 範囲 | ルール |
|---|---|---|
| ハードセーフ | x=`8–92%`, y=`10–90%` | 本体、軸、意味を持つ小物、主要光跡をすべて入れる |
| 主役推奨 | x=`26–74%`, y=`21–79%` | 通常1個のディアボロを置く |
| 周辺演出 | x=`8–92%`, y=`10–90%` | スティック、紐、粒子、背景光だけ |
| 静穏領域 | x=`0–28%`, y=`78–100%` | ラベル対策。細かな意味要素を置かない |

現行UIは画面表示が約270×150（1.80:1）、保存canvas内が592×300（1.97:1）で、どちらも中央 `cover` になる。16:9マスターは保存時に上下が約5%ずつ切られるため、ハードセーフを守る。

### 2.8 背景処理

- 背景は実在場所ではなく、抽象化した無人の競技空間。具体的な体育館、街、学校、国旗、観客席は描かない。
- 3層で統一する。
  1. 最奥: `#050B16`〜`#081A2E` の静かなグラデーション。
  2. 中層: 低コントラストの円弧、グリッド、収束線、スポット光のいずれか1種類。
  3. 前層: カード固有の主光跡と少数の粒子。
- 背景の最も明るい場所は主役の背後または進行方向に置き、主役と競う独立した焦点を作らない。
- 被写界深度は本体が完全に鮮明、中層がわずかに柔らかい、最奥が滑らか、の3段階。強い玉ボケは使わない。

### 2.9 質感・エフェクトの上限

1枚に使えるエフェクトは次の最大3層までとする。

1. **主光跡:** 1本。ディアボロの運動または物語を示す連続した弧。
2. **補助幾何:** 1系統。同心円、収束線、破断線、グリッド、スポット光のどれか。
3. **粒子:** 明瞭に数えられる粒は最大12個。微細な背景粒子を使う場合も画面面積の3%以下。

追加ルール:

- ブルームは発光線の幅の4倍以内。画面全体を霞ませない。
- 稲妻、炎、煙、爆発、液体飛沫、レンズフレアは原則禁止。意味上どうしても必要な場合も補助幾何1系統として扱い、他の補助幾何と併用しない。
- 小物は最大2点、画面面積の合計6%以下。抽象的なリング、欠けた光片、校正目盛りなどに留め、主役より高コントラストにしない。
- 本体の輪郭の75%以上を背景と明確に分離し、光跡や粒子で隠さない。

### 2.10 禁止事項

- 人物、手、腕、顔、目、人体シルエット、観客、人物の影、擬人化。
- ヨーヨー、ベーゴマ、ダンベル、糸巻き、独楽単体、円盤、武器、宇宙船への形状変化。
- 3個以上のカップを1本の軸に付ける誤構造。左右非対称なカップ、余分な軸、余分な紐、余分なスティック。
- 本体へのトゲ、刃、翼、歯車、宝石、傷、ヒビ、ロゴ、刻印、文字の追加。ヒビ演出が必要なカードは背景光または前景の透明層にだけ置く。
- 文字、数字、アルファベット、漢字、疑似文字、ロゴ、ブランド、署名、ウォーターマーク、QRコード。
- カード枠、角飾り、タイトル帯、バッジ、星、レアリティ表示、UI、モックアップ、余白付きポスター。
- 暖色優勢、虹色、金色、セピア、パステル、白背景。クールバイオレット以外の差し色。
- 過密な粒子、全面発光、過度なモーションブラー、強いフィルム粒子、色収差、グリッチ、レンズ汚れ。
- 切れた本体、画面外に出たカップ、極端な接写、主役が画面幅の50%を大きく超える構図。

### 2.11 画角 / 出力仕様

- 制作マスター: `1600 × 900 px`、横長16:9、PNG、sRGB、8bit RGB、背景不透明。
- 生成AIが1600×900を直接出せない場合は、利用可能な横長最大サイズで生成し、全画像を同一手順で中央クロップ後、1600×900へリサイズする。画像ごとの手動トリミングはしない。
- 最終ファイルにはアルファ、EXIF位置情報、生成ツールの可視ウォーターマークを残さない。
- シャープ処理、彩度、コントラスト調整を行う場合は全50枚に同じ数値を適用する。カード別補正は禁止。
- 想定配置: `assets/cards/art_<card-id>.png`。例: `assets/cards/art_sp_worlds.png`。
- 画像ファイル自体にカード名は含めない。ファイル名はコード上のIDだけを使う。

## 3. カード別ニュアンスの設計方法

### 3.1 変更できる5つのレバー

各カードは、固定preambleの後ろに次の5項目だけを追記する。タイトルの単語をそのまま物体化するのではなく、**主比喩1つ + 補助比喩1つ**へ翻訳する。

| 可変レバー | 変えてよい内容 | 変えてはいけない内容 |
|---|---|---|
| ポーズ | 軸の傾き±18°、上下左右の小移動、紐の張り、運動方向、複数使い時の配置 | 本体形状、カメラ種類、極端な遠近、セーフゾーン外への移動 |
| エフェクト | 主光跡1本、補助幾何1系統、粒子0–12個 | 本体改造、上限超過、別パレット、文字に見える記号 |
| 小物 | 最大2点の抽象的・無地の象徴物 | 人物の持ち物、ブランド品、文字入りメダル、主役より目立つ物 |
| 背景の光 | ビーム方向、明暗分布、スポット数、幾何モチーフ | 本体の固定キーライト反転、具体的な会場、観客、国旗 |
| mood | 静謐、緊張、祝祭、粘り、挫折、超越など | 画風、色相セット、材質をmoodに合わせて変更すること |

### 3.2 属性Typeの視覚文法

本体形状を変えず、以下を基本語彙にする。

| Type | ポーズ | 主光跡 / 補助幾何 | 背景光 | mood |
|---|---|---|---|---|
| 高難度型 | 12–18°の強い対角、張った紐 | 狭い高速弧 / 収束するシェブロン | 一方向へ強く収束 | 鋭い集中、圧力、突破 |
| イノベーター型 | 5–14°の意外なオフアクシス | 途中で規則が変わる弧 / 非対称ポリゴン | 左右非対称だが主役は中央 | 好奇心、異質、実験 |
| テクニシャン型 | 0–6°、水平に近く安定 | 正確なループ / 同心円・校正グリッド | 均一で細い | 冷静、精密、反復 |
| ショーマン型 | 5–12°の優雅な上向き弧 | 大きく読みやすい一筆弧 / 扇状スポット | 複数の柔らかな舞台光 | 華、解放、観客不在の祝祭 |
| 万能型 | 0–8°、中央で均衡 | 左右均整の弧 / 円環 | 左右対称 | 自信、安定、王道 |

### 3.3 ランクの視覚文法

ランクは本体色や材質ではなく、エフェクトと背景光の強度だけへ反映する。特別カードはタイトルの意味を優先するが、上限は超えない。

| ランク | 背景コントラスト | 主光跡 | 粒子目安 | moodの強度 |
|---|---:|---:|---:|---|
| S | 高 | 明瞭、長い | 6–12 | 到達、支配、極限 |
| A | 中〜高 | 明瞭 | 5–9 | 熟達、自信 |
| B | 中 | 標準 | 3–7 | 実力、堅実 |
| C | 中〜低 | やや短い | 2–6 | 成長、工夫、親しみ |
| D | 低 | 細く短い | 0–4 | 未熟だが継続、衝動 |
| E | 最低 | 途切れ気味またはごく細い | 0–3 | 青春、空想、始まり |

### 3.4 特別 / 職人カードの扱い

- 特別カードは、実績またはドラマを主比喩にする。王冠、トロフィー、賞状をそのまま置くより、光の高さ、軌道の完成度、遮蔽物の突破などで表現する。
- 職人カードは、ジャンルの運動原理を主比喩にする。水平、垂直、2個、3個以上など、数と軌道を正確にする。
- 1枚に複数実績を盛り込まない。排出条件のうち、タイトルに直結する最上位の意味だけを描く。
- `未完の大器` のヒビは本体ではなく、背後または前景透明層の光の亀裂にする。canon本体を壊さない。
- `双皿の遣い手` はディアボロ2個、`三連の魔術師` は3個を基本とする。「双皿」は1個のディアボロの2カップを指す絵にしない。

## 5. ドリフト抑制の運用フロー

### 5.1 原則

ドリフトは「似せる努力」ではなく、**毎回同じ入力を再現できる仕組み**で抑える。生成済みカードを次のカードの参照にする数珠つなぎは禁止し、常に1枚のroot canonへ戻る。

```text
root canon画像
  ├─ 固定 REFERENCE_LOCK_v1
  ├─ 固定 STYLE_PREAMBLE_A_v1
  └─ カード固有 CARD_DELTA だけ交換
       ├─ art_sp_worlds.png
       ├─ art_sp_expelled.png
       └─ ...全50枚
```

### 5.2 Step 1: neutral canonを先に確定する

作風決定後、物語性の弱い中立画像を4案生成し、1枚だけをroot canonにする。canonはカード本番50枚には数えない。

**canon用差分**

```text
【CANON_DELTA_v1】
シリーズの形状・画風確認用の中立なcanon画像を作る。ディアボロは正確に1個、画像の完全な中央、軸は水平0°、奥行き回転なし。2本の同形スティックは左右下に均整配置し、一本の紐は中央軸の下を通る浅いU字。本体の全形、左右カップの鏡像、外向きリム、中央軸、紐の接触関係が一目で確認できる。主光跡は短く薄いシアンの弧1本、補助幾何は低コントラストの円環1系統、粒子は4個。小物なし。背景はニュートラルなディープネイビーで、左右対称の弱い光。moodは冷静、均整、未命名。特別な実績、ランク、属性、物語を連想させない。
```

canon候補の採用条件:

- 正しい2カップ + 1軸構造で、左右が鏡像。
- スティック2本、紐1本が物理的に読める。
- パレット、線幅、3段塗り、固定照明がStyle Bible通り。
- 主役がセーフゾーン内に入り、左下が静か。
- タイトル、Type、ランクに偏った演出がない。

採用ファイルを `assets/cards/_canon/canon_style_A_v1.png` とし、以後は**リサイズ、再保存、スクリーンショット化していない同一ファイル**を毎回添付する。

### 5.3 Step 2: canonを凍結して記録する

canonと同じ場所に、少なくとも次を記録する。

| 記録項目 | 例 |
|---|---|
| `canon_version` | `A_v1` |
| canonファイル名 / SHA-256 | `canon_style_A_v1.png` / `<hash>` |
| 固定preamble版 | `STYLE_PREAMBLE_A_v1` |
| 参照ロック版 | `REFERENCE_LOCK_v1` |
| 使用サービス / モデル表示名 | 実際のUI表示を転記 |
| 生成日 / 生成セッション | `YYYY-MM-DD` / URLまたは管理番号 |
| 生成サイズ / 最終サイズ | 実値 / `1600×900` |
| 品質・背景などの設定 | UIまたはAPIで指定した値 |
| canonの完全なプロンプト | 省略せず保存 |

seedや参照強度などの設定が利用できる場合は記録するが、それだけを一貫性の根拠にしない。

### 5.4 Step 3: 1カード = 1独立ジョブで生成する

1. 新しい生成ジョブまたは会話を開く。
2. root canonの同一ファイルを添付する。
3. `REFERENCE_LOCK_v1`、固定preamble、対象カードの差分を全文送る。
4. 同じ設定で候補を2〜4枚生成する。
5. 後述のhard gateを満たす候補だけを比較し、1枚を採用する。
6. 修正する場合も「前の画像をもっと派手に」などの省略指示をしない。root canonへ戻り、完全な固定ブロック + 修正済み差分で再生成する。
7. 採用した他カードを新しい参照画像にしない。派生の派生を作らない。
8. カードごとにツール、モデル、品質、縦横比を変更しない。

画像編集モードで局所修正する場合は、文字や余分な物体の除去など最小範囲に限定する。形状、全体照明、画風に関わる失敗は編集で継ぎ足さず、root canonから再生成する。

### 5.5 Step 4: 生成ログを残す

各PNGに対して1行のmanifestまたは同等の記録を持つ。

| フィールド | 内容 |
|---|---|
| `card_id` | `sp_worlds` など `js/cards.js` と一致するID |
| `title` | 検索用。画像には描かない |
| `file` | `assets/cards/art_sp_worlds.png` |
| `canon_version` | `A_v1` |
| `preamble_version` | `STYLE_PREAMBLE_A_v1` |
| `prompt_sha256` | 固定 + 差分を連結した完全プロンプトのhash |
| `source_job` | 生成セッションまたはジョブ識別子 |
| `generated_at` | ISO日付 |
| `qa` | 後述5項目の点数と判定者 |
| `notes` | 再生成理由、既知の軽微な差 |

### 5.6 Step 5: hard gateと採点

次のhard gateを1つでも外した画像は、見栄えが良くても不採用にする。

- 人物、手、人体シルエット、文字、ロゴ、ウォーターマークがゼロ。
- 通常はディアボロ1個。複数使いは指定数どおり。各個体が2カップ + 1軸。
- canonと同じ本体形状、比率、材質、色。追加装飾なし。
- 2本のスティックと一本の紐。紐の経路が破綻していない。
- 重要要素がハードセーフ内。上下5%を切った1.97:1プレビューでも意味が欠けない。
- A案の画風、固定照明、パレット、エフェクト上限内。
- カード固有の主比喩が、タイトルを読まなくても方向性として伝わる。

hard gate通過後、各0〜2点、合計10点で採点し、**9点以上のみ採用**する。

| 項目 | 0 | 1 | 2 |
|---|---|---|---|
| 形状一致 | canonから明確に逸脱 | 軽微な比率差 | 同一モデルに見える |
| 画風一致 | 別メディウム | 線・塗りに軽微な差 | 同一シリーズ |
| 色・光一致 | 色外れ / 光源反転 | 面積配分に差 | 固定パレット・固定光 |
| 構図耐性 | cropで欠損 | やや窮屈 | 両比率で安定 |
| 固有ニュアンス | 誤読 / 無関係 | 弱い | 主比喩が明瞭 |

### 5.7 Step 6: 10枚ごとの横断監査

- canonを左上に固定し、完成済み10枚を同じ大きさで並べたcontact sheetを作る。
- 本体の幅・外径・リム厚・軸長、線幅、背景の明度、シアン面積、粒子密度を横比較する。
- 最も派手な1枚と最も静かな1枚だけを比較せず、中間3枚も確認する。
- 外れが2枚以上見つかった場合は、以後のプロンプトを個別調整する前に、canonまたは固定preambleの解釈が崩れていないか確認する。
- 固定preamble自体を変更した場合は `v2` とし、旧版と混在させない。原則として、全50枚を同じ版で再生成する。

### 5.8 Step 7: モデル更新・長期中断への対処

- 可能な限り同じモデル表示名、品質、サイズ、背景設定を50枚で固定する。
- サービス側のモデル更新、長期中断、担当者交代後は、まず代表3枚（`sp_worlds`、`sp_expelled`、`mx_D_technician`）を同じ入力で再生成し、旧採用品と横断監査する。
- 3枚中1枚でもhard gate不通過、または2枚以上が8点以下なら、本番再開前にcanon版を更新するか、旧環境で続行できる手段を検討する。
- 中断前の最後の生成画像を参照にして再開しない。常にroot canonと保存済み完全プロンプトから再開する。

### 5.9 Step 8: 共通後処理と実装受け渡し

1. 全画像を同じ中央クロップ規則で16:9にする。
2. 同じリサイズ方式で1600×900へ統一する。
3. sRGB / 8bit RGB / 不透明PNGへ統一する。
4. 16:9、1.80:1、1.97:1の3プレビューを作り、主役と意味要素の欠けを確認する。
5. `art_<card-id>.png` で保存し、manifestのIDと一致させる。
6. リポジトリ同梱後、ネットワーク取得ではなくsame-originで読み込めることと、canvas保存時にCORS汚染が起きないことを確認する。
7. 実装ではカードID画像を第一候補、旧Type画像または署名SVGをフォールバックにする。
