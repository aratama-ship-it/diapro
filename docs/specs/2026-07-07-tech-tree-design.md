# 技術解禁ツリー（TR）設計書

- 作成日: 2026-07-07
- 対象: ディアボロ選手育成アプリ（app-dev/diabolo-trainer）
- ステータス: 設計（承認待ち）

## 1. 目的

ジャンル（1D水平/1D垂直/2D/3D以上）を最初から全部練習できる現状を改め、
**基礎ジャンルの習熟が上位ジャンルを解禁していく「技術ツリー」**を導入する。
大学スタートを本当のゼロから積み上げるハードモードにし、育成に段階と達成感を与える。

## 2. 解禁ルール

### 2.1 ツリー構造

```
h1d(1D水平) ──┬─→ v1d(1D垂直)
              └─→ d2(2D) ──→ d3(3D以上)
```

- `h1d`: 根。常に解禁済み。
- `v1d`: `h1d` の習熟 > 20 で解禁（側枝、この先はなし）。
- `d2`:  `h1d` の習熟 > 20 で解禁。
- `d3`:  `d2` の習熟 > 20 で解禁。

### 2.2 指標

- 「習熟」= `genreAvg`（そのジャンルの difficulty/novelty/control 3マスの平均、既存の contest.js 関数）。
- 閾値判定は **厳密に > 20**（習熟ちょうど20は未解禁、20を超えたら解禁）。

### 2.3 データ定義（data.js）

```js
SKILL_TREE: {
  h1d: { requires: null },
  v1d: { requires: { genre: 'h1d', threshold: 20 } },
  d2:  { requires: { genre: 'h1d', threshold: 20 } },
  d3:  { requires: { genre: 'd2',  threshold: 20 } }
}
```

### 2.4 判定関数（contest.js）

```js
// genreId が現在解禁されているか。requires が null なら常に true。
function isGenreUnlocked(state, genreId) {
  const req = DT.DATA.SKILL_TREE[genreId].requires;
  if (!req) return true;
  return genreAvg(state, req.genre) > req.threshold;
}
```

スキルは練習・イベントで上がる一方（減少しない）ため、解禁状態は単調に増加する。
永続フラグを持たなくても、都度計算で常に正しい。

## 3. 初期値と経歴

### 3.1 大学（ハード）

- 技術12マス（4ジャンル×3技術）= **すべて0**。
- 演技構成 = **少し残す**（技術より小さいレンジ。初期案 3〜12、実測で微調整）。
- 結果として大学は h1d のみ解禁で開始し、h1d を20超まで育てて初めて v1d/d2 が解禁される。

### 3.2 他の経歴（高校/中学/幼少期）

- 初期値は**現状維持**（BACKGROUNDS のレンジそのまま）。
- 解禁は 2.1 のルールに従う。h1d の初期習熟が20以下なら v1d/d2 も最初はロックされる
  （例: 2Dの習熟が15なら3Dだけ練習不可）。経歴が上がるほど初期解禁が広い自然な挙動になる。

### 3.3 生成の分離（state.js / data.js）

- 現状 `newCharacter` は技術12マスと演技構成に同一の `bg.statMin/statSpread` を使う。
- 演技構成用の任意フィールド `compMin`/`compSpread` を BACKGROUNDS に追加。
  未指定の経歴は従来どおり `statMin/statSpread` を流用（挙動不変）。
- 大学のみ `statMin:0, statSpread:0`（技術0）＋ `compMin/compSpread`（演技構成を少し残す）を設定。

## 4. UI（app.js）

### 4.1 スロット選択

- 未解禁ジャンルのボタンは `disabled`。ラベルまたは補助表示に解禁条件を出す
  （例: 「🔒 1D水平の習熟20で解禁」）。
- 前月スロットのプリフィルは、スキルが単調増加のため一度解禁したジャンルは維持され安全。

### 4.2 コンディション欄

- 次の解禁目標を表示（例: 「次の解禁：2ディアボロ（1D水平の習熟あと3）」）。
- 全解禁済みなら非表示。

### 4.3 大会エントリー

- 未解禁ジャンルの部門は選択不可（練習できない部門には出場できない）。
- `maxEntries` による上限とは別に、解禁済みジャンルのみを候補にする。

## 5. 解禁演出と状態（engine.js / state.js）

### 5.1 検知タイミング

- **練習直後**に告知する。`applyTraining` の最後で、現在解禁されている全ジャンルを算出し、
  `state.announcedUnlocks`（告知済みジャンルidリスト）に無いものを新規解禁として
  結果メッセージに追加（例: 「2ディアボロが解禁された！」）、リストへ追加する。
- イベントでスキルが上がって解禁された場合も、`announcedUnlocks` 未登録なら次の練習結果で
  一度だけ告知される（単調性により重複告知は起きない）。

### 5.2 状態追加

- `state.announcedUnlocks`（新規フィールド）。**キャラ生成時点で既に解禁済みのジャンルで初期化**する
  （h1d は常時解禁、経歴によっては v1d/d2 等も初期解禁）。これにより「開始時から使えるジャンル」は
  告知されず、プレイ中に新しく解禁されたものだけが告知される。
- 保存バージョン **v8 → v9**。`OLD_KEYS` に v8 を追加し、旧セーブはクリア。

## 6. テスト方針

### 6.1 新規単体テスト

- `isGenreUnlocked`: h1d常時true、v1d/d2はh1d習熟20境界、d3はd2習熟20境界。
- 大学の `newCharacter`: 技術12マス=0、演技構成>0。
- スロット選択で未解禁ジャンルが選べない（disabled）。
- 解禁演出が新規解禁時に一度だけ出る（`announcedUnlocks` 重複防止）。
- 大会エントリー候補から未解禁ジャンルが除外される。

### 6.2 既存バランステストの再検証

- `tests/test-simulation.js` の `pickCombo` を「**解禁済みジャンルの最弱マス**を狙う」に修正
  （現状は全12マスから argmin。未解禁マスを選ばないようにする）。
- 高校スタートで h1d 習熟が20未満の場合、まず h1d を育てる挙動になり、
  v5バランス調整（国内大会 base/sd）で得た分布が変わる可能性がある。
- 卒業率・順位系・世界大会出場系の各テストを再実行し、破綻すれば閾値20やバランス値を微調整。
  関連: [[project_diabolo_trainer_balance_v5]]。

## 8. 非対象（YAGNI）

- ジャンルごとの解禁「イベント一枚絵」や凝った演出（テキスト告知のみ）。
- 解禁ツリーの可視化画面（将来のRC=レーダーチャート等と別途検討）。
- 解禁閾値の経歴別カスタム（まずは全経歴共通の20で通す）。
