## 0. 結論

### 結論1
**360度の情報は役に立つ。**  
特に「前」「横」「後ろ」「3/4前」「3/4後ろ」の情報があると、後から
- 後ろ向き
- 横向き
- 振り向き
- 走る
- 座る
- 表情違い
- 小物追加
- コマ割り漫画化

などの指示は通りやすくなる。

### 結論2
ただし、**最初からフル360度データだけを大量に作るのは、効率が最良とは限らない。**

OpenAIの画像生成ガイドでは、複数画像を使って生成・編集できる一方で、**大量の画像よりも少数の高品質な参照画像の方が扱いやすい**とされている。また、各画像の役割を順番で明示することが推奨されている。  
出典: OpenAI「Creating images with ChatGPT」  
- “You can upload multiple images to guide generation or editing, but a small set is usually easier to manage than a large one.”  
- “Refer to each image by order and explain how they relate to each other.”  
https://openai.com/academy/image-generation/

### 結論3
もっと効率の良い中核は、**360度ターンアラウンド単体ではなく、キャラクターバイブル（設定書）＋基準画像セット**を作ること。

つまり必要なのは次の3層。

1. **視覚基準**
   - 代表画像
   - ターンアラウンド
   - 表情集
   - ポーズ集
   - 衣装資料

2. **言語基準**
   - 何がそのキャラらしさか
   - 絶対に変えてはいけない要素
   - 変えてよい要素
   - 禁止事項

3. **運用基準**
   - どの順番で画像を作るか
   - どう修正するか
   - どう参照するか
   - どうプロンプト化するか

## 1. OpenAI公式情報から分かること

OpenAIの画像生成ガイドでは、GPT Image系モデルで**画像の生成と編集が可能**であり、**1枚または複数の画像を参照画像として使って新しい画像を作れる**と案内されている。  
出典: OpenAI API「Image generation」  
https://developers.openai.com/api/docs/guides/image-generation

また、OpenAI Academyのガイドでは、編集時には
- 何を変えるか
- 何を変えないか

を明確に指示することが有効だとされている。  
具体的には、  
“Change only X. Keep everything else exactly the same.”  
のような明示的な制約が、精密な編集に有効と説明されている。  
出典: OpenAI「Creating images with ChatGPT」  
https://openai.com/academy/image-generation/

さらに、OpenAIの「Introducing ChatGPT Images 2.0」では、
- multi-scene continuity
- character reference sheet
- consistent identity variation

のような例が示されており、**連続したシーンでの一貫性**や**キャラクター基準シート**の考え方と相性が良い。  
出典: OpenAI「Introducing ChatGPT Images 2.0」  
https://openai.com/index/introducing-chatgpt-images-2-0/

## 2. 何を作るべきか

### 目的
写真から人物または猫などをデフォルメ化し、今後も
- いろいろな角度
- いろいろな表情
- いろいろなポーズ
- ギャグ表現
- 漫画
- スタンプ
- グッズ
- 解説図
- バナー
- 動画素材

へ再利用できるようにする。

### 目標
- 毎回顔が変わらない
- 服が勝手に変わらない
- 服が消えない
- 耳が消えない
- しっぽが勝手に増減しない
- 体型が勝手に変わらない
- 年齢感が変わらない
- 目の大きさや位置が大きくぶれない
- 作風が毎回ズレない

## 3. 360度情報は本当に必要か

### 結論
**必要だが、完全な360度全方位セットを初手で作る必要はない。**

### 必須度の高い角度
まずは以下で十分。

1. 正面
2. 右側面
3. 真後ろ
4. 左側面

この4方向を標準4面図として固定すると、左右非対称の要素まで確認できる。3/4前と3/4後ろは4面承認後の拡張とする。

### 追加すると強いもの
- 3/4前
- 3/4後ろ
- 上向き気味
- 俯瞰気味
- あおり気味
- 座り姿
- 走り姿
- 手を広げた全身
- 無表情の基準立ち姿

### 理由
画像モデルは「見えていない部分」を推測して描くため、後ろ姿や横顔を一度も基準化していないと、
- 服の背面デザインが変わる
- 髪型の後頭部が変わる
- 耳やしっぽの位置が変わる
- 背中の模様が消える

といった問題が起きやすい。

### ただし
参照画像の数が増えすぎると、管理が難しくなる。OpenAIのガイドでも、少数の明確な参照画像の方が扱いやすいとされている。  
そのため、**大量の360度資料を毎回全部渡すより、「使う目的に応じて少数の基準画像を選んで渡す」運用の方が現実的。**

## 5. 推奨するキャラ資産の構成

### Layer 1: Core Identity Sheet
そのキャラの核。

#### 含めるもの
- 名前
- 種別（人間 / 猫 / 動物 / マスコット）
- 年齢感
- 性格キーワード
- デフォルメ比率
- 身長比率（頭身）
- 顔の特徴
- 目の形
- 鼻の形
- 口の形
- 耳の形
- 髪型 / 毛並み
- 体型
- シルエット
- 基本色
- 服装
- 装飾
- 持ち物
- 禁止事項

#### 例
- 頭身: 2.5頭身
- 顔: 丸顔
- 目: 大きめ、黒目がち、横長ではなくやや丸
- 鼻: 小さい三角
- 耳: 必ず存在、先端は丸くしない
- 服: 赤いパーカー、白いひも、黒いパンツ
- シルエット: 上半身やや大きめ、足短め
- 禁止: 耳を消す、服を別デザインにする、パーカーを脱がせる

---

### Layer 2: Visual Canon Set
視覚的な正解集。

#### 必須
1. 正面・右側面・背面・左側面の4面図
2. 4面を切り出した角度別画像
3. 顔アップ
4. 全身シルエット
5. 表情4種
   - 無表情
   - 笑顔
   - 驚き
   - 怒り or 困り

#### あると強い
- 歩き
- 走り
- 座り
- ジャンプ
- 手を振る
- 物を持つ
- うしろから見た時のフード・背面柄
- 靴の形
- しっぽの付け根

---

### Layer 3: Costume Lock Sheet
服装固定用。

#### 含めるもの
- 正面・側面・背面の衣装
- 服の各パーツ名
- 色指定
- 模様
- ロゴの有無
- ボタン数
- ファスナー有無
- 紐の本数
- ポケット位置
- 靴
- 帽子
- アクセサリー

#### 例
- 赤パーカー
- フード付き
- 白い紐2本
- 左胸に小さなワッペン1つ
- 袖口は黒
- ズボンは黒の短パン
- 白ソックス
- グレーのスニーカー

#### 禁止欄
- シャツに変えない
- 帽子を追加しない
- 裸足にしない
- ロゴの位置を変えない
- パーカーを勝手に脱がせない

---

### Layer 4: Style Lock Sheet
作風固定用。

#### 含めるもの
- 線の太さ
- 線の揺れ
- 塗りの方式
- 影の数
- ハイライトの入り方
- 輪郭の丸さ
- 色数
- テクスチャ
- 背景の扱い
- 目の描き方
- デフォルメの度合い

#### 例
- 線は均一でやや太め
- 塗りはフラット
- 影は1段だけ
- ハイライトは小さく1点
- 目の輪郭は黒で明確
- 口は簡略化
- 背景は基本なしまたは単色
- ぬいぐるみ風ではなくイラスト風
- かわいすぎる幼児化はしない

---

### Layer 5: Expression & Motion Sheet
「笑わせる」「泣かせる」「怒らせる」などの表情・演技の基準。

#### 含めるもの
- 喜
- 怒
- 哀
- 楽
- 困る
- ドヤ顔
- 照れ
- 驚き
- 爆笑
- 泣き笑い
- 真顔

#### 動作
- 手を振る
- 走る
- うしろを向く
- 振り返る
- 飛び上がる
- 座る
- 寝転ぶ
- 拍手
- 指差し
- 物を持つ

## 6. 一番効率のいい制作フロー

### Phase 1: 元資料の整理
#### 人物の場合
- 顔写真
- 全身写真
- 横顔写真
- 後ろ姿写真
- 髪型が分かる写真
- よく着る服 or 固定衣装資料

#### 猫の場合
- 顔正面
- 全身
- 横向き
- 背面
- しっぽの形
- 肉球や耳など特徴部位
- 模様が分かる写真

### 注意
ここでは枚数より、**特徴が分かるか**が重要。

---

### Phase 2: デフォルメ方針の決定
まず決めるべきこと。

- 何頭身か
- どれくらい丸めるか
- 顔の比率はどれくらい誇張するか
- リアル寄りか、かわいい記号寄りか
- 線画か、塗り込みか
- スタンプ向きか、漫画向きか、グッズ向きか

#### ここで決めると良い軸
- 2頭身
- 2.5頭身
- 3頭身
- 目を大きくする度合い
- 鼻を記号化するか
- 手足の短さ
- 服の簡略度

---

### Phase 3: 最初の基準立ち絵を作る
最初に作るべきは「いろんな表情」ではなく、**無表情の正面基準立ち絵**。

#### 理由
ここが全ての起点になるため。

#### この段階で固めるもの
- 顔の比率
- 目の位置
- 耳の位置
- 髪型 / 毛並み
- 体型
- 服装
- 色
- シルエット

---

### Phase 4: ターンアラウンド生成
正面基準が決まったら、次を作る。

- 正面
- 右側面
- 真後ろ
- 左側面

この4面を標準ターンアラウンドとする。3/4前と3/4後ろは、4面の承認後に必要に応じて追加する。

#### 方針
この段階ではポーズを付けすぎない。  
**回転情報の確定が目的**であり、演技や表情はまだ後回し。

---

### Phase 5: 表情セット
無表情基準を壊さない範囲で、表情差分を作る。

#### 先に作るべきもの
- 微笑み
- 笑顔
- 困り
- 驚き
- 怒り
- 泣き
- ドヤ顔

---

### Phase 6: 動作セット
ここでようやく
- 後ろ向き
- 振り向き
- 走る
- ジャンプ
- 物を持つ

などを作る。

---

### Phase 7: 運用テンプレート化
ここが重要。

「キャラの画像」を作るだけで終わらせず、**今後の指示テンプレート**を作る。

- 種別
- 頭身
- 顔の比率
- 目の形
- 耳の形
- 鼻の形

- 基本配色
- 服装
- 靴
- アクセサリー
- しっぽ
- 模様
- 性格トーン

- 表情

## 10. ChatGPT Images 2.0での実践的な考え方
### 1. いきなり完成キャラ量産をしない
まずは**基準画像の制作**に専念する。

### 2. 生成より編集を多用する
OpenAIのガイドでも、既存画像を編集する際は、変える部分と維持する部分を明確にするとよいとされている。  
そのため、安定運用では「ゼロから毎回生成」より「基準画像から差分編集」の方が有利。

### 3. 毎回、画像の役割を明示する
OpenAIのガイドでは、複数画像を使う場合、順番に応じて各画像の役割を説明することが推奨されている。  
例:
- Image 1 = キャラ正面基準
- Image 2 = 服装基準
- Image 3 = 表情基準
- Image 4 = 作風基準

### 4. 参照画像は少数精鋭
大量に渡すのではなく、**目的に対して必要最小限**に絞る。

### 5. 「何を変えないか」を毎回書く
例:
- このキャラクターの顔、頭身、服装、耳、しっぽ、模様、色、線画スタイルは固定
- ポーズと表情だけ変える
- 背景だけ変える

## 11. 具体的なプロンプト設計の考え方

### 基本形
```text
このキャラクターを、同じ作風・同じ頭身・同じ顔立ち・同じ服装で維持したまま描いてください。
変えてよいのはポーズと表情だけです。
耳、目の形、服のデザイン、配色、体型、しっぽ、線画のタッチは変更しないでください。
```

### 後ろ向き
```text
このキャラクターを後ろ向きで描いてください。
背面の服装デザイン、頭身、シルエット、耳の位置、しっぽの形は基準画像と一致させてください。
作風は完全に同じにしてください。
新しい服や装飾を追加しないでください。
```

### 笑わせる
```text
このキャラクターを大笑いしている表情で描いてください。
表情だけを変え、顔の基本構造、頭身、服装、配色、耳、髪型、しっぽ、作風は維持してください。
```

### 服を変えない強い指定
```text
このキャラクターの服は固定です。
服を脱がせないでください。
服のデザイン、色、形、ロゴ、紐、ポケット、靴を変更しないでください。
ポーズだけを変えてください。
```

#### Identity Layer
- 顔
- 頭身
- 種別
- シルエット
- 性格印象

#### Costume Layer
- 服
- 靴
- 小物

#### Style Layer
- 線
- 塗り
- 色
- 影

#### Expression Layer
- 表情
- 目線
- 口の開き

#### Pose Layer
- 体の向き
- 手足
- アクション

この5層に分けると、  
「ポーズだけ変える」「表情だけ変える」「背景だけ変える」が管理しやすい。

## 14. さらに強い発想
### 将来、本当に量産するなら「キャラ聖書」を作る

単なる設定資料ではなく、次をまとめた運用文書。

- 代表画像
- 仕様書
- 禁止事項
- 角度資料
- 衣装資料
- 表情資料
- 小物資料
- 推奨プロンプト
- 禁止プロンプト
- 失敗例
- 修正方針

### 失敗例を集める意義
何がブレやすいかを先に知れる。

例:
- しっぽが消えた
- 服が変わった
- 目が別キャラになった
- 線が太くなりすぎた
- 子供っぽくなりすぎた

これを
- よくある崩れ
- 再修正プロンプト

として蓄積していく。

## 16. 今回の問いへの直接回答

### Q1. 360度からの情報をデータとして作る必要はあるか
**ある。かなり有効。**  
特に後ろ姿、横姿、3/4方向の安定性には効く。

### Q2. それがあれば、後に指示は通りやすくなるか
**通りやすくなる。**  
ただし、それだけでは不十分で、  
- 固定要素
- 禁止事項
- 服装仕様
- 作風仕様
- 表情仕様

までセットで持っていた方が強い。

### Q3. もっといいアイデアはあるか
**ある。**  
「360度資料単体」より、  
**キャラ仕様書 + 基準画像パック + 差分編集運用**  
の方が効率がよい。

### Q4. 勝手に服が変わる、耳が消えるなどを防ぐには
最も効くのは以下。

1. 正面・側面・背面の基準画像
2. 服装ロックシート
3. 禁止事項の明文化
4. 毎回「変えてよいもの / 変えてはいけないもの」を書く
5. 新規生成より基準画像からの差分編集
6. 参照画像を少数精鋭にする
7. 失敗例を蓄積して修正プロンプト化する
